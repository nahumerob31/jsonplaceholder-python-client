# Contributing Guidelines

Thank you for considering contributing to **jsonplaceholder-python-client**!

## How to Contribute
1. **Fork the repository** and clone it locally.
2. **Create a new branch** for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Write code** and include tests where applicable.
4. **Run the test suite** to ensure everything passes:
   ```bash
   pytest
   ```
5. **Commit your changes** with clear, descriptive messages.
6. **Push** the branch to your fork and open a **Pull Request** against the `main` branch.

## Code Style
- Follow PEP 8 guidelines.
- Use type hints for public functions.
- Run `flake8` and `black` before submitting.

## Reporting Issues
- Use the issue tracker to report bugs or suggest enhancements.
- Provide a clear description and, if possible, a minimal reproducible example.
